How it works?

Html code is simple,just keep the right hierarchy.

<ol class="increment-ul">
    <li>第一章</li>
    <li>第二章
            <ol>
                    <li>第一节</li>
                    <li>第二节
                            <ol>
                                <li>第一段</li>
                                <li>第二段</li>
                                <li>第三段
                                    <ol>
                                        <li>第一小段</li>
                                        <li>第二小段</li>
                                        <li>第三小段</li>
                                </ol>
                                </li>
                        </ol>
                    </li>
                    <li>第三节</li>
            </ol>
    </li>
    <li>第三章
            <ol>
                    <li>第一节</li>
                    <li>第二节</li>
                    <li>第三节</li>
            </ol>
    </li>
</ol>

The css code

.increment-ul,.increment-ul li {list-style: none;margin:0;padding:0;}
.increment-ul li:before {display:inline-block;margin-right: 0.5em;font-size: 1.6em;font-style: italic;color:#999;}

.increment-ul {counter-reset:l1;}
.increment-ul>li:before {content:counter(l1)'.';counter-increment:l1;}

.increment-ul>li>ol {counter-reset:l2;}
.increment-ul>li>ol>li:before {content:counter(l1)'.'counter(l2);counter-increment:l2;}

.increment-ul>li>ol>li>ol {counter-reset:l3;}
.increment-ul>li>ol>li>ol>li:before {content:counter(l1)'.'counter(l2)'.'counter(l3);counter-increment:l3;}

.increment-ul>li>ol>li>ol>li>ol {counter-reset:l4;}
.increment-ul>li>ol>li>ol>li>ol>li:before {content:counter(l1)'.'counter(l2)'.'counter(l3)'.'counter(l4);counter-increment:l4;}
/*you can add more level here*/

You can add more level yourself.l1-l4（is just variables you named） reset the level of li tag.

mac去掉menu菜单

最后更新: Wed Sep 10 2014 12:39:11 GMT+0800

Mac 默认把自己和应用程序的菜单（menu）永远放在最上方。menu+dock（去dock简单，直接点右键 trun hideing on） 太占地方了，决心去掉。

  1. 下载安装 PlistEdit Pro

  2. Finder 打开 Applications，右键一个软件，比如 sublime，show Package cotents，找到 Info.plist，右键用 PlistEdit Pro 打开。New Child 新建一个键值叫 Application UI Presentation Mode，class 为 number，value 为 4（也就是 All suppressed）。可能回提问是否解锁该文件，当然，不然怎么改。

3.重启 sublime。

以后，切换到 sublime 的时候，类似全屏状态，鼠标靠近屏幕上方出菜单，下方出 dock。这个桌面干净了,你可以对常用的软件都这么干。

另：传说中的 magicmenu 在 lion 下报错。

Array every 方法

array.every(callback[, thisObject])

Tests whether all elements in the array pass the test implemented by the provided function.根据提供的函数测试 是否 所有数组元素都满足该条件

a=[1,2,3,4,5,6,7];
alert(a.every(function(el,i,arr){return el>0}));

找到 false 就立即（退出循环）返回 false，否则返回 true。下面例子返回 1－5

console.log(a.every(function(el,i,arr){console.log(el);return el<5})) <="" pre="">


回调的三个参数：element, index, array


every 不改变 array.


空数组 满足任意条件，甚至不可能的 >1 && <0


console.log([].every(function(el,i,arr){console.log(el);return el>1 && el<0})) <="" pre="">


The range of elements processed by every is set before the first invocation of callback. Elements which are appended to the array after the call to every begins will not be visited by callback. If existing elements of the array are changed, their value as passed to callback will be the value at the time every visits them; elements that are deleted are not visited.第一次调用前，数组元素就已经被处理。新添加或者修改数组元素，没用了。看下面例子


a=[1,2,3,4,5,6,7]
console.log(a.every(function(el,i,arr){a[0]=-1;console.log(el);return el>0}))
1... 7 true
console.log(a)
[-1, 2, 3, 4, 5, 6, 7]

驾驶如何省油

最后更新: Wed Sep 10 2014 12:37:50 GMT+0800

看看美国鬼子如何解释这个问题：How to Save Gas with Eight Secret Tips，和简体中文世界看到的不太一样

  • 关掉空调，把窗户打开。
  • 快没油了，再加。（你又不是去戈壁沙漠，前面会有加油站的）
  • 加油枪倒过来，控一下。（难了，据我所知，除了新泽西，美国加油站都是自助，自己动手加油刷卡。国内，似乎都是工作人员才能动加油枪）
  • 邮箱盖子拧紧。（“汽”油挥发很快）
  • 开慢点，使用经济车速。每个车不太一样，看说明书，一般80-100km/h
  • 不要没事变道。（个人体验，在美国变道很少，经常开了100公里，前后左右还是那几辆车，打死也不变道。国内，似乎到处都是f1赛场）
  • 高速路用巡航，路况复杂的时候别用。
  • 能关掉的都关掉（音响之类）。
  • 按时换机油、滤清器。
  • 不要急停急驶。
  • 清理车内不需要带的东西。

二维生物

最后更新: Wed Sep 10 2014 12:37:50 GMT+0800

霍金在《时间简史》中提到为什么没有二维生物

二维空间似乎不足以允许像我们这样复杂生命的发展。例如，如果二维动物吃东西时不能将之完全消化，则它必须将其残渣从吞下食物的同样通道吐出来；因为如果有一个穿通全身的通道，它就将这生物分割成两个分开的部分，我们的二维动物就解体了。

反对的声音：

  1. 变形虫 - 变形虫通过伪足将猎物包起来，产生一个食物泡，然后食物泡与其细胞内的初级溶酶体融合形成次级溶酶体，将食物消化。

  2. flat land

    在平面國，所有一切都看起來像線。屋子大多是五邊形的，以避免過度尖銳的角刺傷人。平面國的人民則依照階級分成多種幾何圖形。以男性而言，士兵和低階工人是等腰三角形，中等階級則是等邊三角形。專業的男性和紳士是正方形和五邊形，上流的貴族階級則是六邊形和更多邊的形狀。最後，最高級的教士是圓的階級（其實，他們只是有著上百個邊，以致於每個邊過短到幾乎感覺不出來，因此和真正的圓相當接近，嚴格來說他們只是幾百邊形）

    如同線國國王不能了解二維，正方形也不能理解三維，球只好訴諸最後手段：把正方形帶到空間國。正方形終於了解了除了長、寬以外還有高的維度，他進而興奮的向圓提出四維的類比概念：在一維中，移動的點與二端點產生線；二維中，移動的線與四端點產生正方形；三維中，移動的正方形與八端點產生立方體。那麼四維中，立方體往某個方向移動就能產生十六個端點的超立方體。球斥責之為胡言亂語，讓激動的正方形回到了平面國。

    via 1884年出版的《平面国》Flatland

  3. 这个理由是不成立的，因为即便是三维物体，也不是连续的统一体，物质、物质之间总是有空隙的，从绝对的角度，三维的人也是一堆彼此有空隙的粒子组成的.二维的人，只要有一个特定的力场，那就可以把两半维系在一定距离，仍旧是一个完整的人

Mac 优化

最后更新: Wed Sep 10 2014 12:39:11 GMT+0800

Mac 和我

Mac 用了三年，主板撒啤酒换掉了（淘宝上海二手，2500,mac店要8500），硬盘不关机到处走，坏道若干，成天4国。后来把硬盘直接分成两半分区，使用后面那个，基本上不太死机。现在用的是朋友龚骚淘汰的ps3的硬盘。找朋友换的主板，笔记本本来就紧凑，mac更是密密麻麻，换完之后，电量显示、声卡麦克风都挂了。Mac 就象当初买的 swatch，一次性消费品，基本上没有升级、维修的必要和可能性。

这次的 mac lion 使用了将近一年，没有重装，居然还能用，奇迹。当然，这和我现在不太乱装软件有关。当初用 ubuntu desktop 大约半年肯定挂，windows 大约1年。毕竟 mac 下的软件要比 linux 和 windows 下普遍质量高。

Mac 优化项目（lion）

  • Menu > System Preferences > Users & Groups > Login Items,删除不需要的启动项。

  • 关掉蓝牙、语音识别和共享。web share 是 apache，file share 是 afp 和 smb(windows 局域网，开始－运行－\macip 访问),internet share 分享互联网，类似wingate的代理服务器，甚至可以模拟安卓的热点功能)

  • 关掉 Widgets

  • 用 Activity Monitor 检查还有什么不需要的劳什子

  • 时不时用磁盘工具，verify 一下权限。这个很重要，*nix 和 windows 系统很不一样，权限在文件本身而不是象windows那样在其它地方其中管理。

  • 桌面少放点东西。我个人的恶习是：过几天建一个日期格式的（2013-05-02）目录，然后把所有桌面东西拖进去。好处是：桌面很清爽，要找的东西肯定在这个目录里。坏处是：几乎找不到！（2013-05-02/2013-04-18/2013…）还好，一般要找的东西，大约都是个把月内的，时间长的还不如上网找。逼急了，用 find。

  • 关掉 dock 效果。（再美的老婆，也架不住天天看啊）

  • 如果你晚上总是关机，运行一下那些 dayli,weekly,monthly 脚本,清理垃圾文件。人家mac设计的是 － 永远不关机，所以这些系统自带脚本都是半夜运行。

    sudo periodic daily weekly monthly

    查看一下执行结果

    ls -al /var/log/*.out

    输出大致如下：

    -rw-r—r— 1 root wheel 496994 5 Mar 03:15 /var/log/daily.out

    -rw-r—r— 1 root wheel 2088 1 Mar 06:02 /var/log/monthly.out

    -rw-r—r— 1 root wheel 5289 5 Mar 03:15 /var/log/weekly.out

值得商榷

  • time machine 用过一阵，很不实用。绝大部分时候，我需要的只是备份 百十个 文本类的东西，和几个较大的软件安装包和一些电影。我不需要整个系统备份！直到现在，一个1年前的 time machine 仍然占据着我的一个移动硬盘。好吧，也许在世界毁灭之前，我可能还会用到它。有鉴于此，我现在手工拷贝一些东西到移动硬盘，再小的东西，自己给自己发邮箱里。

  • 系统升级。无数的说法推荐及时升级，我怀疑。就个人体验来看，好处没体会到，而且 mac 升级动辄2－3G，跟重装有什么区别？有两次升级带来了麻烦，一次四国，还有一次不能启动，然后 ctrl+shift+command+电源 键恢复 smc。我也不太理解这有什么用处，但就是稀里糊涂，能启动了。我不是 mac support 或者测试，我买电脑为了用，我可不想成天打断手头的事，去给 mac 去做什么测试工作。

  • *nix 和 windows 文件系统的习惯不一样。windows c d e f，重装只需要（甚至不需要）格式化一个盘符，装系统很方便。所以我的 windows 机器一般有4－5个系统（+ubuntu），只要硬盘不完蛋，今天我就能干活。而 *nix 官方说法是 mount /home 到独立的分区。我个人土办法是：弄一个 fat分区，作为备份区，大家（windows,ubuntu,mac）都往这里备份。Mac 有一点挺恶心（人家是为了安全，ubuntu 安装的时候如果选了加密 home 也是这个德性），即便硬盘拆下来，装移动硬盘盒，挂到其它电脑上，很多目录不让进！没权限！弄得跟 FBI 似的。

  • 第三方优化工具，如 Monolingual 删除多余的语言。Onyx 清理缓存等文件。FanControl 控制风扇。我个人不喜欢装这些东西。早期，用这些类似软件，导致系统不稳定。在

    150公里/小时+不那么安全 VS. 80公里/小时+安全

    之间，我选择 安全。
