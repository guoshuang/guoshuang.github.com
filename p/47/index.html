Mac 优化

最后更新: Wed Sep 10 2014 12:39:11 GMT+0800

Mac 和我

Mac 用了三年，主板撒啤酒换掉了（淘宝上海二手，2500,mac店要8500），硬盘不关机到处走，坏道若干，成天4国。后来把硬盘直接分成两半分区，使用后面那个，基本上不太死机。现在用的是朋友龚骚淘汰的ps3的硬盘。找朋友换的主板，笔记本本来就紧凑，mac更是密密麻麻，换完之后，电量显示、声卡麦克风都挂了。Mac 就象当初买的 swatch，一次性消费品，基本上没有升级、维修的必要和可能性。

这次的 mac lion 使用了将近一年，没有重装，居然还能用，奇迹。当然，这和我现在不太乱装软件有关。当初用 ubuntu desktop 大约半年肯定挂，windows 大约1年。毕竟 mac 下的软件要比 linux 和 windows 下普遍质量高。

Mac 优化项目（lion）

  • Menu > System Preferences > Users & Groups > Login Items,删除不需要的启动项。

  • 关掉蓝牙、语音识别和共享。web share 是 apache，file share 是 afp 和 smb(windows 局域网，开始－运行－\macip 访问),internet share 分享互联网，类似wingate的代理服务器，甚至可以模拟安卓的热点功能)

  • 关掉 Widgets

  • 用 Activity Monitor 检查还有什么不需要的劳什子

  • 时不时用磁盘工具，verify 一下权限。这个很重要，*nix 和 windows 系统很不一样，权限在文件本身而不是象windows那样在其它地方其中管理。

  • 桌面少放点东西。我个人的恶习是：过几天建一个日期格式的（2013-05-02）目录，然后把所有桌面东西拖进去。好处是：桌面很清爽，要找的东西肯定在这个目录里。坏处是：几乎找不到！（2013-05-02/2013-04-18/2013…）还好，一般要找的东西，大约都是个把月内的，时间长的还不如上网找。逼急了，用 find。

  • 关掉 dock 效果。（再美的老婆，也架不住天天看啊）

  • 如果你晚上总是关机，运行一下那些 dayli,weekly,monthly 脚本,清理垃圾文件。人家mac设计的是 － 永远不关机，所以这些系统自带脚本都是半夜运行。

    sudo periodic daily weekly monthly

    查看一下执行结果

    ls -al /var/log/*.out

    输出大致如下：

    -rw-r—r— 1 root wheel 496994 5 Mar 03:15 /var/log/daily.out

    -rw-r—r— 1 root wheel 2088 1 Mar 06:02 /var/log/monthly.out

    -rw-r—r— 1 root wheel 5289 5 Mar 03:15 /var/log/weekly.out

值得商榷

  • time machine 用过一阵，很不实用。绝大部分时候，我需要的只是备份 百十个 文本类的东西，和几个较大的软件安装包和一些电影。我不需要整个系统备份！直到现在，一个1年前的 time machine 仍然占据着我的一个移动硬盘。好吧，也许在世界毁灭之前，我可能还会用到它。有鉴于此，我现在手工拷贝一些东西到移动硬盘，再小的东西，自己给自己发邮箱里。

  • 系统升级。无数的说法推荐及时升级，我怀疑。就个人体验来看，好处没体会到，而且 mac 升级动辄2－3G，跟重装有什么区别？有两次升级带来了麻烦，一次四国，还有一次不能启动，然后 ctrl+shift+command+电源 键恢复 smc。我也不太理解这有什么用处，但就是稀里糊涂，能启动了。我不是 mac support 或者测试，我买电脑为了用，我可不想成天打断手头的事，去给 mac 去做什么测试工作。

  • *nix 和 windows 文件系统的习惯不一样。windows c d e f，重装只需要（甚至不需要）格式化一个盘符，装系统很方便。所以我的 windows 机器一般有4－5个系统（+ubuntu），只要硬盘不完蛋，今天我就能干活。而 *nix 官方说法是 mount /home 到独立的分区。我个人土办法是：弄一个 fat分区，作为备份区，大家（windows,ubuntu,mac）都往这里备份。Mac 有一点挺恶心（人家是为了安全，ubuntu 安装的时候如果选了加密 home 也是这个德性），即便硬盘拆下来，装移动硬盘盒，挂到其它电脑上，很多目录不让进！没权限！弄得跟 FBI 似的。

  • 第三方优化工具，如 Monolingual 删除多余的语言。Onyx 清理缓存等文件。FanControl 控制风扇。我个人不喜欢装这些东西。早期，用这些类似软件，导致系统不稳定。在

    150公里/小时+不那么安全 VS. 80公里/小时+安全

    之间，我选择 安全。

玫瑰和月季的区别

最后更新: Wed Sep 10 2014 12:37:50 GMT+0800

简单说：没有区别！

玫瑰 英文 rose ；月季 英文 chinese rose,我的理解是：圣女果和西红柿，狼和狗的区别。小资们要郁闷了。

月季(种：月季 R. chinensis)、玫瑰（玫瑰 R. rugosa）和蔷薇 是同 蔷薇科 蔷薇属 的姊妹花。（界 门 纲 目 科 属 种，还记得吗？）蔷薇 － 一種蔓藤爬籬笆的小花

维基百科的资料

月季（學名：Rosa chinensis），又名月月红（江苏浙江）、月月花、长春花（吉林长春）、庚申蔷薇，原產於中國中部的贵州、湖北、四川等地，現遍布世界各地。

玫瑰、月季和薔薇都是薔薇屬植物，之間有種類上的區別，沒有科屬上的差異。在漢語中人們習慣把花朵直徑大、單生的品種稱為玫瑰或月季，小朵叢生的稱為薔薇。但在英語中它們均稱為rose。

玫瑰原产是中国。在古時的漢語，「玫瑰」一詞原意是指紅色美玉。後來，一種野生薔薇屬植物Rosa rugosa，因果實又大又紅豔，因而以「玫瑰」作為稱呼。在日本Rosa rugosa稱為浜茄子，朝鮮稱為海棠花。

玫瑰原產地 - 中國山东、甘肃、昆明、遼寧、河北等。

日常生活中的玫瑰 - 在欧洲诸语言中，蔷薇、玫瑰、月季都是一个词，如英语是rose，德语是die Rose，因为蔷薇科植物从中国传到欧洲后他们并不能看出它们的不同之处，但实际上这是不同的花种。今日世界上最常見所的雜交玫瑰（Rosa hybrida，亦稱現代薔薇或現代月季）是數百年來由許多種薔薇花及月季花育種雜交所誕生的成品。

今天在日常用语中，“玫瑰”已成為多種薔薇屬植物的通稱。玫瑰依目前正式登記的品種，約有三萬左右。

hexo显示fontawesome问题

最后更新: Wed Sep 10 2014 12:37:50 GMT+0800

hexo 自带的 fontawesome 版本低，少了一些“字”。但很奇怪，chrome 显示正常，而 firefox 显示空的框子。经过折腾，终于解决问题：

原来，我曾经把 fontawesome（老版本的） 字体安装到系统字体了！在这种情况下，chrome 优先从网络 @font-face 引用字体，而 firefox 则是操作系统的字体优先！

删除系统中的 fontawesome，然后替换 themes/light/css/font 中的字体，重新 generate。

hexo 程序本身的字体在：

/usr/local/lib/node_modules/hexo/assets/themes/light/source/css/font

替换这里的字体，以后 init 新项目以后就不用每次在 themes/light/css/font 折腾了。

arguments 是个“伪“数组，是个 object

An Array-like object corresponding to the arguments passed to a function.The arguments object is not an Array. It is similar to an Array, but does not have any Array properties except length. via mdn

function sayHi() {
  var a = arguments.shift() // error! arguments is not Array
  alert(a)
}
sayHi()

错误！（Object has no method ‘shift’）arguments 是个“伪“数组，是个 object

(function() {

区分 Array Date 类型 [[Class]]

object,array 都是 object 类型，如何区分呢？

alert( typeof {key:'val'} ) // Object is object
alert( typeof [1,2] ) // Array is object
alert( typeof new Date ) // Date object

Fortunately, there is a hidden [[Class]] property in all JavaScript native objects. It equals “Array” for arrays, “Date” for dates etc.

var toClass={}.toString;
alert( toClass.call( [1,2] ) ) // [object Array]
alert( toClass.call( new Date ) ) // [object Date]
  1. Copy a reference to toString for objects into toClass variable.
  2. Call it, but provide the object which we are going to check as this. So, object toString becomes applied to arrays, dates etc. We have to do it, because their own toString behaves differently. For arrays it returns element list, for dates it returns the string representation.

该方法对原始类型也可以

alert( toClass.call(123) ) // [object Number]
alert( toClass.call("primitive")) ) // [object String

用 instanceof 检测数组

注意：跨窗口（frame）不行！每个window有自己的 层次结构。用 [[class]] 方法

Although, it will not work if arr was created in another window or iframe and then passed into current window. That’s because each window has it’s own object hierarchy.
To workaround that, most frameworks use [[Class]] for native objects.
